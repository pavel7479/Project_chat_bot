1. User query
2. Retriever → достаёт знания (docs)
3. SessionMemory → достаёт историю (history)
4. ContextBuilder → собирает всё вместе
5. LLM → ОДИН раз генерирует ответ

✔️ Нормальная схема
запрос пользователя
      ↓
Retriever.search()
      ↓
docs (база знаний)
      ↓
SessionMemory.get_history()
      ↓
history (диалог)
      ↓
ContextBuilder.build()
      ↓
готовый контекст
      ↓
LLM.generate()
      ↓
ответ

Как теперь работает система:
1. Пользователь задаёт вопрос
2. Retriever ищет информацию в базе (docs)
3. IntentClassifier определяет намерения (query + история)
4. Берём историю диалога (history)
5. ContextBuilder собирает всё
6. LLM генерирует ответ

1. Пользователь → ChatService
2. ChatService → FaissRetriever (находит релевантные документы)
3. ChatService → Контекст (история + найденные документы)
4. ChatService → LLM генерации ответа (формирует текст ответа)
